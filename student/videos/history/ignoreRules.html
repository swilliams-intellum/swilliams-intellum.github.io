A full list of ignore rules can be seen in URLChangeRules.swift.

If any of these rules return true the state change is not handled by the mobile app and is left to the SPA. This means a new web view is not pushed onto the navigation stack.

The URL Path is the Same

The main rule to concern yourself with is the SamePathRule. If all that is changing is the query string or the fragment, we do not push a new web view onto the stack.

For example, the following URL changes do not produce a push navigation.

Query Parameters

/path/to/page?tab=about
↓
/path/to/page?tab=privacy

Fragment Changes

/path/to/page#about
↓
/path/to/page#privacy
